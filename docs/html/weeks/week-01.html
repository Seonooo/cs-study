OSI/TCP-IP 계층 모델

network 완료 slug: osi-tcpip

OSI 7계층과 TCP/IP 4계층을 비교하며, 패킷이 어떻게 캡슐화되어 네트워크를 흐르는지 추적한다. "URL 입력 시 일어나는 일"을 계층 관점으로 설명할 수 있게 되는 것이 목표.

학습 목표

  1. 두 모델이 왜 다르게 나뉘었는지 설명
  2. "URL 입력 시 일어나는 일"을 7계층 관점으로 추적
  3. L4 vs L7 로드밸런서의 차이
  4. HTTP/3가 UDP 위에 만들어진 이유

핵심 키워드

  • OSI 7계층
  • TCP/IP 4계층
  • 캡슐화 (encapsulation)
  • PDU
  • L4/L7 로드밸런서
  • HTTP/3 (QUIC)

딥다이브

  • 계층별 PDU — Frame(L2) → Packet(L3) → Segment(L4) → Data(L5–7). 헤더가 쌓이고 벗겨지는 메커니즘.
  • URL 추적 시나리오 — 브라우저에서 URL 입력 → DNS 조회 → TCP 연결 → TLS 핸드셰이크 → HTTP 요청/응답 → 렌더링. 각 계층에서 일어나는 일.
  • L4 vs L7 LB — L4는 IP/Port 기반, L7은 HTTP 헤더/경로 기반. 처리량과 라우팅 정밀도의 트레이드오프.
  • HTTP/3 = QUIC over UDP — TCP HoL blocking 문제를 UDP 위에서 신뢰성 직접 구현으로 해결.

토론 주제

OSI 모델은 실무에서 정말 의미가 있는가? TCP/IP 모델로 충분하지 않나?

면접 단골 Q&A

OSI와 TCP/IP 모델의 차이는?

OSI는 ISO의 prescriptive 7계층 표준 어휘, TCP/IP는 IETF RFC 1122의 descriptive 4계층 실구현. OSI L5–L7이 TCP/IP의 Application 한 계층에 묶임.

"URL 입력 시 일어나는 일"을 계층 관점으로 설명해 주세요.

DNS 조회(L7) → TCP 3-way handshake(L4) → TLS(L5/6) → HTTP 요청(L7) → 라우팅(L3) → MAC 프레임 전송(L2) → 응답 역순.

L4와 L7 로드밸런서의 선택 기준은?

처리량·저지연이 우선이면 L4, 경로/헤더 기반 라우팅·SSL 종료·세션 친화가 필요하면 L7.
