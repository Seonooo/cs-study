PHASE 3 · DATABASE · WEEK 11

관계형 모델과 정규화

database slug: relational-normalization

1NF~BCNF가 어떤 이상 현상을 방지하는지 함수 종속성 관점에서 이해하고, OLTP/OLAP에 따라 정규화/반정규화 정책이 어떻게 갈리는지 설명할 수 있어야 한다.

학습 목표

  1. 정규화 단계와 방지하는 이상 현상
  2. 함수 종속성 분석
  3. OLTP / OLAP별 정규화 정책
  4. 분산 환경 키 전략 선택

핵심 키워드

  • 1NF ~ BCNF
  • 함수 종속성
  • OLTP vs OLAP
  • UUID v7
  • Snowflake ID

딥다이브

  • 이상 현상 — 삽입 / 갱신 / 삭제 이상. 정규화는 이걸 구조적으로 제거.
  • 3NF까지의 직관 — "키, 키의 전부, 키만"이 종속 대상. BCNF는 결정자가 모두 키여야 함.
  • OLTP vs OLAP — OLTP는 3NF 표준(쓰기 일관성). OLAP는 별 스키마 등 의도적 비정규화(읽기 성능).
  • 분산 PK — auto-increment는 단일 노드 병목. UUID v7은 시간 정렬 가능, Snowflake는 (time + node + seq)로 64비트.

토론 주제

실무에서 3NF까지만 가는 이유는? 언제 반정규화를 할 것인가?

면접 단골 Q&A

정규화의 목적과 단계는?

중복 제거로 갱신/삭제 이상 방지. 1NF(원자값) → 2NF(부분 종속 제거) → 3NF(이행 종속 제거) → BCNF(결정자가 모두 후보키).

언제 반정규화를 하나요?

읽기 빈도가 압도적으로 높고 조인 비용이 병목일 때. 단, 갱신 시점의 정합성 보장 책임이 애플리케이션으로 옮겨감을 인지.

분산 환경에서 PK 전략은?

auto-increment는 중앙 시퀀스 병목 → UUID v7(시간 정렬 + 인덱스 친화) 또는 Snowflake(시간+노드+시퀀스, 정렬 + 짧음).
