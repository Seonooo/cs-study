PHASE 5 · OPS · WEEK 20 · 신설

CI/CD·Docker·옵저버빌리티·시스템디자인 베이직

ops 신설 주차 slug: devops-observability

커밋부터 프로덕션까지의 흐름, Docker가 VM과 다른 이유, 옵저버빌리티 3 시그널, 그리고 10배 트래픽을 흡수하는 시스템 설계의 기초를 한 주에 통합 정리한다. 2026 면접에서 신입에게도 묻기 시작한 영역.

학습 목표

  1. 커밋부터 프로덕션까지 CI/CD 흐름
  2. Docker가 VM과 다른 점, 이미지 레이어
  3. 옵저버빌리티 3 시그널 (logs / metrics / traces)과 correlation ID
  4. 캐시·MQ·수평확장으로 10배 트래픽 흡수 설계
  5. 프로덕션 장애 대응 (탐지 → 완화 → RCA → 재발 방지)

핵심 키워드

  • CI/CD 파이프라인
  • Docker / 컨테이너
  • Logs / Metrics / Traces
  • 캐싱 · 메시지 큐 · 수평 확장
  • 10x 트래픽

딥다이브

  • CI/CD 흐름 — 커밋 → 자동 테스트 → 빌드(아티팩트/이미지) → 스테이징 배포 → 프로덕션 롤아웃(Rolling/Canary/Blue-Green).
  • Docker vs VM — VM은 게스트 OS 전체, 컨테이너는 호스트 커널 공유 + 네임스페이스/cgroups로 격리. 이미지는 레이어 캐시로 빌드/배포 효율.
  • 옵저버빌리티 3 시그널 — Logs(구조화 JSON + correlation ID), Metrics(시계열 수치, 알람), Traces(요청이 서비스 경계를 넘는 경로 추적).
  • 10x 트래픽 설계 — 캐시(읽기 부하 흡수), 읽기 복제본, 비동기 큐(쓰기 평탄화), 수평 확장 + 로드밸런싱, 레이트 리밋.
  • 장애 대응 — 탐지(알람/SLO) → 완화(롤백/스케일아웃) → 근본 원인 분석 → 재발 방지(포스트모템).

토론 주제

신입~주니어가 운영 경험 없이 옵저버빌리티 질문에 어떻게 답할 것인가?

면접 단골 Q&A

CI와 CD의 차이는?

CI는 커밋 시 자동 테스트·빌드로 통합 안정성 확보. CD는 그 산출물을 스테이징/프로덕션까지 자동 배포(Delivery는 수동 승인, Deployment는 완전 자동).

Docker 컨테이너와 VM의 차이는?

VM은 하이퍼바이저 위에 게스트 OS를 통째로. 컨테이너는 호스트 커널을 공유하고 네임스페이스/cgroups로 격리 → 가볍고 빠른 시작.

트래픽이 10배로 늘면 어떻게 대응하나요?

캐시로 읽기 흡수, 읽기 복제본 추가, 쓰기는 메시지 큐로 평탄화, 무상태 서비스 수평 확장 + 로드밸런싱, 레이트 리밋으로 보호. 병목을 측정 후 단계적으로.
