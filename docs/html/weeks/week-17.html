Spring MVC·AOP·Filter/Interceptor

framework 신설 주차 slug: spring-mvc-aop

요청이 DispatcherServlet에 도착해 응답이 나가기까지의 전체 흐름을 추적하고, 횡단 관심사를 Filter·Interceptor·AOP 중 어디에 둘지 판단할 수 있어야 한다.

학습 목표

  1. DispatcherServlet 요청 처리 흐름
  2. AOP 핵심 용어와 동작 원리
  3. JDK Proxy vs CGLIB 차이
  4. Filter / Interceptor / AOP 적용 시점 비교
  5. @RestController vs @Controller

핵심 키워드

  • DispatcherServlet (Front Controller)
  • HandlerMapping · ViewResolver
  • AOP (Aspect · Advice · Pointcut · JoinPoint)
  • JDK vs CGLIB Proxy
  • Filter vs Interceptor vs AOP

딥다이브

  • 요청 흐름 — DispatcherServlet → HandlerMapping(핸들러 탐색) → HandlerAdapter(실행) → Controller → ViewResolver / MessageConverter → 응답.
  • AOP 5요소 — Aspect(모듈), JoinPoint(적용 지점), Pointcut(선택 표현식), Advice(실행 코드: Before/After/Around), Weaving(적용).
  • JDK vs CGLIB — 인터페이스 있으면 JDK Dynamic Proxy, 없으면 CGLIB(상속 기반). Spring Boot는 기본 CGLIB.
  • self-invocation 함정 — 같은 클래스 내부 메서드 호출은 프록시를 거치지 않아 AOP/@Transactional 미적용.
  • Filter vs Interceptor vs AOP — Filter(서블릿 레벨, 요청 전체) → Interceptor(스프링 MVC, 핸들러 전후) → AOP(메서드 레벨, 비즈니스 로직).

토론 주제

횡단 관심사(로깅·인증·트랜잭션)를 Filter / Interceptor / AOP 어디에 둘 것인가?

면접 단골 Q&A

DispatcherServlet의 요청 처리 흐름을 설명해 주세요.

DispatcherServlet이 모든 요청을 받아 HandlerMapping으로 핸들러를 찾고, HandlerAdapter로 컨트롤러를 실행, 반환값을 ViewResolver/MessageConverter로 처리해 응답.

Filter와 Interceptor의 차이는?

Filter는 서블릿 컨테이너 레벨로 DispatcherServlet 이전·이후. Interceptor는 스프링 MVC 레벨로 핸들러 실행 전후, 스프링 빈/컨텍스트 접근 가능.

AOP가 같은 클래스 내부 호출에 안 먹는 이유는?

Spring AOP는 프록시 기반. this.method() 자기 호출은 프록시를 우회하므로 advice가 적용되지 않음. 자기 호출 분리 또는 AspectJ weaving으로 해결.
