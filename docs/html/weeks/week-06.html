PHASE 2 · OS · WEEK 06

프로세스와 스레드, 스케줄링

os slug: process-thread

프로세스와 스레드가 자원을 어떻게 공유·분리하는지, 컨텍스트 스위칭이 왜 비용이 크고 어디서 발생하는지, 그리고 리눅스 CFS 같은 현대 스케줄러의 동작 원리를 짚는다.

학습 목표

  1. 프로세스 vs 스레드 자원 공유 관점
  2. 컨텍스트 스위칭 비용 구성
  3. 스케줄링 알고리즘 (FCFS, SJF, RR, MLFQ) 비교
  4. 리눅스 CFS 동작 원리
  5. IPC 방식별 선택 기준

핵심 키워드

  • PCB / TCB
  • 컨텍스트 스위칭
  • FCFS · SJF · RR · MLFQ
  • 리눅스 CFS
  • IPC (파이프 · 소켓 · 공유메모리 · 시그널)

딥다이브

  • PCB가 담는 것 — PID, 레지스터, 스택/힙 포인터, 파일 디스크립터 테이블, 시그널 핸들러, 스케줄링 상태.
  • 컨텍스트 스위칭 비용 — 직접: 레지스터 저장/복원. 간접: 캐시·TLB 무효화로 인한 미스 폭증.
  • CFS의 핵심 — vruntime이 가장 작은 태스크 우선. red-black tree로 O(log N) 선택. nice 값으로 가중치 조정.
  • IPC 방식별 — 파이프(부모-자식, 단방향) · 소켓(분산) · 공유메모리(최고 속도, 동기화 필요) · 시그널(비동기 통지).

토론 주제

멀티프로세스 vs 멀티스레드 — 언제 무엇을 쓸 것인가?

면접 단골 Q&A

프로세스와 스레드의 차이는 무엇인가요?

프로세스는 독립된 메모리 공간(코드·데이터·힙·스택)과 OS 자원을 보유. 스레드는 같은 프로세스 안의 실행 흐름으로 코드·데이터·힙을 공유하고 스택만 분리. 통신 비용·격리 수준의 트레이드오프.

컨텍스트 스위칭 비용은 무엇으로 구성되나요?

레지스터 저장/복원의 직접 비용 + 캐시·TLB 무효화로 인한 간접 비용. 후자가 더 크게 작용하는 경우가 많음.

IPC 방식을 어떻게 선택하나요?

속도가 최우선이면 공유메모리 + 락. 분산이면 소켓. 단순 부모-자식 통신이면 파이프. 비동기 통지가 필요하면 시그널.
