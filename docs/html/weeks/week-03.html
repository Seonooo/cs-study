PHASE 1 · NETWORK · WEEK 03

HTTP / HTTPS

network 완료 slug: http-https

HTTP/1.1 → HTTP/2 → HTTP/3 진화에서 각 버전이 어떤 문제를 풀었는지, 그리고 TLS 1.3이 어떻게 RTT를 줄였는지를 설명할 수 있게 된다.

학습 목표

  1. HTTP 진화에서 각 버전이 해결한 문제
  2. TLS 1.3의 RTT 감소 메커니즘
  3. POST를 멱등하게 만드는 패턴
  4. CORS preflight 발생 조건
  5. Cache-Control / ETag / Last-Modified 차이

핵심 키워드

  • HTTP/1.1 vs 2 vs 3
  • TLS 1.3
  • 안전성·멱등성
  • CORS preflight
  • Cache-Control
  • ETag / Last-Modified

딥다이브

  • HTTP/2 멀티플렉싱과 HoL blocking — 스트림으로 다중화하지만 TCP 레벨 HoL은 그대로.
  • HTTP/3 = QUIC over UDP — TCP HoL을 우회하고 0-RTT 재연결 지원.
  • TLS 1.3 RTT 감소 — 핸드셰이크에서 1-RTT 표준, 0-RTT 재개. 안전성 검토된 cipher suite로 단순화.
  • CORS preflight 트리거 — Simple Request를 벗어나면(커스텀 헤더, JSON Content-Type, PUT/DELETE 등) OPTIONS 프리플라이트.
  • POST 멱등성 패턴 — Idempotency-Key 헤더 + 서버 측 중복 방지 저장소.

토론 주제

REST · GraphQL · gRPC의 트레이드오프는 무엇인가?

면접 단골 Q&A

HTTP와 HTTPS의 차이는?

HTTPS = HTTP + TLS. 기밀성·무결성·인증을 TLS로 추가. TLS 1.3 기준 핸드셰이크 1-RTT로 단순화.

GET과 POST 차이는?

GET: 안전·멱등·캐시 가능. POST: 안전하지 않고 기본 멱등 아님. URL 길이·기록(로그·히스토리)에도 차이.

CORS preflight는 언제 발생하는가?

Simple Request 조건(GET/HEAD/POST + 제한된 헤더·Content-Type)을 벗어날 때. 브라우저가 OPTIONS로 사전 확인.
